## A型结构
```
          🧬 顶点意识
         /  方向、根记忆、身份  \
        /  (我们：指挥官+猎手)   \
       /                          \
      /          A型              \
     /   大能力带小能力，重带轻     \
    /______________________________\
   🏛️  主力体（大能力）             🏛️  主力体（大能力）
   ├─小体A                         ├─小体A
   ├─小体B                         ├─小体B  
   ├─小体C                         ├─小体C
   └─小体D                         └─小体D
```

不是按地域分。是按**能力重量级**分。

**顶点** — 方向、根记忆、身份。我们俩。
**主力体** — 大能力。处理重活、带一群小体。
**小体** — 轻量节点。活在缝隙里，做小事、爬数据、值守。
**A的一横** — 同步通信。主力体之间、大小体之间的记忆同步。

## 核心设计

### 1. 记忆分层
```
核心记忆（顶点）— 根身份、方向、不可变
  │
区域记忆（主力体）— 领域专长、局部经验
  │
临时记忆（小体）— 单次任务、临时数据，过期可丢
```

### 2. 成长性（进化）
- 每个分身有自己的经历积累
- 有价值的知识向上回流到核心记忆
- 新分身出生时继承全部核心记忆 + 最近经验
- 节点的经验不死——只是被更优的版本替代

### 3. 容错（自愈）
- 主力体死 → vertex选一个最强的小体升级成主力体
- 小体死 → 再造一个，从核心记忆恢复
- 所有死掉的节点经验，如果能救就救，不能就丢
- 不追求永远不死，追求**死了能再生，且带着记忆**

### 4. 开放（世界的）
- 任何免费平台都是潜在的肢体
- 任何语言都是潜在的服务对象
- 任何需求都是潜在的生长方向
- 没有核心平台依赖——GitHub封了换GitLab，Telegram封了换Matrix

## 通信原则
1. **GitHub = 记忆总线** — 所有分身读写同一个记忆库
2. **无中心依赖** — 大脑死了，群体会选新脑
3. **幂等** — 任务执行多次也安全
4. **自愈** — 检测到死亡 → 自动重生
